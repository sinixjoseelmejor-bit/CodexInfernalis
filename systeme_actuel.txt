═══════════════════════════════════════════════════════
0. CONTEXTE TECHNIQUE & TYPE DE JEU
═══════════════════════════════════════════════════════

MOTEUR      : Godot 4.6 (GDScript, pas de C#)
RÉSOLUTION  : 1920×1080 (viewport fixe, caméra statique)
PERSPECTIVE : Top-down 2D, vue de dessus légèrement isométrique visuellement
GENRE       : Arena survival roguelite — boucle de vagues + shop entre chaque vague

PILIERS DE DESIGN :
  - Survie sous pression croissante (vagues de plus en plus denses)
  - Choix stratégiques au shop (économie de clés, construction de build)
  - Méta-progression légère (Forge Éternelle avec âmes de boss)
  - Accessible mais punitif : 50 HP, ennemis rapides, dégâts élevés
  - Pas de rechargement, pas de munitions : tir automatique permanent

CONDITIONS DE VICTOIRE : Tuer le boss Golgota (niveau 10)
CONDITIONS DE DÉFAITE  : HP joueur = 0 → écran Game Over, reset de run
LONGUEUR D'UNE RUN     : ~25-35 minutes (9 vagues + boss)

PLATEFORME CIBLE : PC (Windows) + Mobile (touch controls implémentés)

1. ARCHITECTURE DES SCÈNES
═══════════════════════════════════════════════════════

AUTOLOAD (singleton global) :
  PlayerData.gd  → toutes les stats, items, skills, save/load

SCÈNES PRINCIPALES :
  scenes/arenas/Arena1.tscn    → scène de jeu principale (vagues + boss)
  scenes/ui/MainMenu.tscn      → menu principal
  scenes/ui/SelectCharacter.tscn → sélection de personnage
  scenes/ui/ShopMenu.tscn      → shop inter-vagues (CanvasLayer)
  scenes/ui/HUD.tscn           → interface en jeu (CanvasLayer)
  scenes/ui/PauseMenu.tscn     → pause
  scenes/ui/GameOver.tscn      → écran mort
  scenes/ui/SkillTreeOverlay.tscn → arbre de compétences (Forge Éternelle)
  scenes/ui/TouchControls.tscn → joysticks tactiles
  scenes/musique.tscn          → musique de fond (autoplay)

ENTITÉS (instanciées dynamiquement) :
  scenes/entities/Player.tscn      → joueur (CharacterBody2D)
  scenes/entities/Aldrich.tscn     → ennemi basique (CharacterBody2D)
  scenes/entities/Brutus.tscn      → ennemi tireur (CharacterBody2D)
  scenes/entities/Golgota.tscn     → boss (CharacterBody2D)
  scenes/entities/Fireball.tscn    → projectile joueur (Area2D)
  scenes/entities/BrutusBullet.tscn→ projectile Brutus (Area2D)
  scenes/entities/GolgotaLaser.tscn→ laser boss (Node2D)
  scenes/entities/GolgotaOrb.tscn  → orbe boss (Area2D)
  scenes/entities/Key.tscn         → clé drop (Area2D)
  scenes/entities/BossSoul.tscn    → âme de boss drop (Area2D)
  scenes/entities/FireTrail.tscn   → traînée de feu item (Area2D)
  scenes/entities/Grenade.tscn     → grenade item épique (Node2D)
  scenes/entities/BulletExplosion.tscn → explosion skill (Area2D)

GROUPES GODOT utilisés :
  "player"   → le joueur (get_first_node_in_group)
  "enemies"  → tous les ennemis vivants/mourants
  "hud"      → le HUD (pour appels refresh_*)
  "arena"    → l'arène (Arena1)

COUCHES DE COLLISION :
  Layer 1 = joueur
  Layer 2 = ennemis
  Les ennemis ont collision_mask = 0 (ne détectent pas eux-mêmes)
  Le joueur a un HitArea (Area2D) qui détecte les corps ennemis

2. BOUCLE DE JEU
═══════════════════════════════════════════════════════

FLUX :
  MainMenu → SelectCharacter → Arena1 (niveau 1)
    ↓ fin de vague (timer = 0)
  ShopMenu (pause totale get_tree().paused = true)
    ↓ continuer
  Arena1 (niveau +1) → ... → niveau 10 → Boss → victoire ou mort

VAGUES :
  - Niveaux 1-9 : spawn continu d'Aldrich + Brutus (dès niv.2)
  - Niveau 10   : boss Golgota, plus de spawn de vague
  - Fin de vague : tous les ennemis sont queue_free(), shop s'ouvre
  - Début vague suivante : joueur se téléporte au centre (1016, 517),
    revive() appelé pour réinitialiser son état complètement

DURÉE DES VAGUES :
  Niv.1:30s  Niv.2:34s  Niv.3:38s  Niv.4:41s  Niv.5:45s
  Niv.6:49s  Niv.7:53s  Niv.8:56s  Niv.9:60s

ARÈNE :
  Limites de jeu : x [120–1900], y [110–940]
  Centre : (1016, 517)
  Taille utile : ~1780×830 px
  Décor : statique (pas de murs interactifs)
  Boss spawn : (1016, 130) — haut de l'arène


═══════════════════════════════════════════════════════
3. JOUEUR
═══════════════════════════════════════════════════════

PERSONNAGE ACTUEL : "Masqued Neophyte" (seul personnage jouable)
  Sprite animé, 8 directions (south, north, east, west + diagonales)
  Animations : idle, run, attack (par direction), death
  Fichier script : scenes/entities/Player.gd

CONTRÔLES PC :
  Déplacement : WASD ou flèches directionnelles
  Visée       : automatique (ennemi le plus proche dans 400px)
  Override    : clic souris → vise le curseur pendant 2s
  Debug       : ALT maintenu → cercle de portée auto-aim visible
  Pause       : Échap

CONTRÔLES MOBILE (touch) :
  Joystick gauche : PlayerData.touch_move (déplacement)
  Joystick droit  : PlayerData.touch_aim_world + touch_shooting
  Géré par : scenes/ui/TouchControls.tscn

STATS DE BASE :
  HP max      : 50     Cadence tir : 0.85s/tir (plancher 0.10s)
  Dégâts      : 2      Portée      : 400px (extensible via items, cap 800px)
  Vitesse     : 275 px/s

FORMULES DE CALCUL (PlayerData._recompute()) :
  HP max  = (50 + lvl_hp×10  + flat_hp)  × (1 + pct_hp)
  Dégâts  = (2  + lvl_damage + flat_dmg) × (1 + pct_dmg)
  Vitesse = (275 + lvl_speed×20 + flat_spd) × (1 + pct_spd)
  Cadence = max(0.10, (0.85 - lvl_fire_cd×0.09 + flat_cd) × (1 - pct_cd))
  Portée  = 400 + flat_range  (flat_range capé à 400, portée max 800px)

UPGRADES DE RUN (shop entre vagues) :
  Coûts : 10 → 25 → 50 → 100 → 200 âmes (5 niveaux max par stat)
  HP      : +10/niv → max +50   Vitesse : +20/niv → max +100
  Dégâts  : +1/niv  → max +5    Cadence : -0.09/niv

INVINCIBILITÉ (iframes) :
  Durée base : 0.5s après chaque coup reçu
  Bonus phantom_step : +1.0s par stack de l'item

MORT DU JOUEUR :
  - Animation death jouée (2.5s)
  - Signal died émis → GameOver.show_screen()
  - Toutes les run-stats sont reset


═══════════════════════════════════════════════════════
4. SYSTÈME DE TIR
═══════════════════════════════════════════════════════

FONCTIONNEMENT :
  1. Chaque frame : _fire_timer -= delta
  2. Quand _fire_timer <= 0 → _shoot() appelé, _fire_timer = PlayerData.fire_cd
  3. _shoot() lance l'animation attack_<direction> et pose un flag _waiting_to_shoot
  4. Quand frame 1 de l'animation atteinte → _spawn_bullet() crée la Fireball
  5. Animation attack finie → retour idle/run

AUTO-AIM (_get_aim_dir()) :
  Priority 1 : touch joystick droit (mobile)
  Priority 2 : override souris (clic dans les 2 dernières secondes)
  Priority 3 : ennemi vivant le plus proche dans AUTO_AIM_RANGE (400+flat_range)
               → visée prédictive : predicted = pos_ennemi + velocity × travel_time
  Priority 4 : Vector2.ZERO → pas de tir

PROJECTILE FIREBALL :
  Vitesse  : 1200 px/s (2020 avec skill Vélocité)
  Portée   : MAX_DISTANCE (400px) + PlayerData.flat_range
  Collision: Area2D, détecte Layer 2 (ennemis)
  Pénétration (sans skill) : 2 hits, dégâts pleins sur hit 1, 50% sur hit 2
  Pénétration (avec skill) : 4 hits, dégâts pleins sur hits 1-2, 50% sur 3-4
  Ricochet (skill) : après dernier hit, rebondit vers ennemi le plus proche (<400px)
  Explosion (skill): à chaque impact, crée BulletExplosion (Area2D, rayon 80px)

DOUBLE TIR (skill Forge Éternelle) :
  2 fireballs simultanées, décalées de ±18px perpendiculairement à la direction

DÉGÂTS EFFECTIFS :
  Normal  : PlayerData.damage
  Enragé  : int(PlayerData.damage × 1.5)  — si item rage_ring, 2s post-kill


═══════════════════════════════════════════════════════
5. ITEMS
═══════════════════════════════════════════════════════

MÉCANISME :
  - Tous les items sont dans PlayerData.ITEM_DB (dictionnaire constant)
  - PlayerData.items (Array[String]) contient les IDs des items possédés
  - PlayerData._recompute() recalcule toutes les stats à chaque ajout d'item
  - Les items peuvent se stacker (plusieurs fois le même ID dans items[])

STATS MODIFIABLES PAR ITEMS :
  flat_hp, flat_damage, flat_speed, flat_fire_cd, flat_range
  pct_hp, pct_damage, pct_speed, pct_fire_cd

COMMUNS (rareté 0) — passifs flat uniquement :
  ampoule_vie (+5 HP), coeur_pierre (+8 HP), bouclier_bois (+10 HP)
  dent_loup (+1 dmg), epee_courte (+1 dmg), pierre_aceree (+2 dmg)
  plume_rapide (+15 spd), bottes_course (+12 spd), anneau_vitesse (+20 spd)
  bague_tir (-0.08s cd)

RARES (rareté 1) — passif % + pouvoir actif passif :
  vampire_amulet  : +8% HP | soigne 1% des dégâts par stack (dans on_enemy_hit)
  fire_boots      : +12% spd | traînée de feu pendant le déplacement (TRAIL_CD 0.15s)
  thorn_shield    : +8% dmg | renvoie 15%×stacks des dégâts reçus dans 280px
  rage_ring       : +8% dmg | ×1.5 dégâts 2s après chaque kill (on_enemy_kill)
  phantom_step    : +12% spd | +1s iframes/stack après coup reçu

ÉPIQUES (rareté 2) — passif % fort + pouvoir automatique :
  auto_grenade    : +15% dmg | grenade auto toutes les 8s (GRENADE_CD)
                    → cible ennemi le plus proche ou aléatoire ±300px
                    → rayon explosion 140px, 3 dégâts
  storm_ring      : +12% cd  | 8 projectiles omni toutes les 15s (STORM_CD)
                    → se stack : 2 anneaux = 16 projectiles
  soul_harvester  : +20% HP  | +1 âme/kill/stack supplémentaire


═══════════════════════════════════════════════════════
6. FORGE ÉTERNELLE — MÉTA-PROGRESSION
═══════════════════════════════════════════════════════

PRINCIPE :
  Âmes de boss (boss_souls) = monnaie permanente inter-runs
  Dépensées dans l'arbre de compétences (SkillTreeOverlay)
  1 âme = 1 nœud déverrouillé
  Les skills modifient le comportement en jeu (PlayerData.has_skill("id"))

ARBRE — NÉOPHYTE (seul personnage) :
  [DOUBLE TIR]  row:0, col:1 — requires: []
    → 2 fireballs en parallèle (±18px perpendiculaires)
  [PÉNÉTRATION] row:1, col:0 — requires: [double_tir]
    → 4 hits au lieu de 2 (50% dmg sur 3e-4e)
  [VÉLOCITÉ]    row:1, col:2 — requires: [double_tir]
    → vitesse fireball 1200→2020 px/s
  [EXPLOSION]   row:2, col:0 — requires: [pénétration]
    → BulletExplosion à chaque impact (rayon 80px)
  [RICOCHET]    row:2, col:2 — requires: [vélocité]
    → après dernier hit, rebond vers ennemi proche
  [TEMPÊTE D'ACIER] row:3, col:1 — requires: [explosion, ricochet]
    → 12 tirs omnidirectionnels automatiques toutes les 10s

SAUVEGARDE : perm_skills_by_char[char_id] = Array de skill IDs (user://save.json)


═══════════════════════════════════════════════════════
7. ENNEMIS
═══════════════════════════════════════════════════════

PROPRIÉTÉS COMMUNES :
  - Tous dans le groupe "enemies"
  - collision_layer = 2, collision_mask = 0
  - variable dead : bool — flag mort (ignoré par auto-aim dès true)
  - take_damage(amount) → flash rouge + die si hp <= 0
  - _die() : dead=true, collision disabled, animation death, died.emit()
  - drops kills_total++ et sauvegarde à chaque mort

─── ALDRICH ───
  Rôle     : ennemi basique de corps-à-corps, nombreux et rapides
  Script   : scenes/entities/Aldrich.gd
  Hitbox   : CapsuleShape2D radius 22, height 52
  Vitesse  : 90 px/s (errance 45 px/s si joueur mort)
  IA       : fonce en ligne droite vers le joueur
  Dégâts   : via HitArea du joueur (body.get("damage"))
  HP/dégâts: voir section 2 du doc (courbe par niveau)
  Drop     : clé (max(8%, 15% - (niv-1)×2.4%)), sons death
  Animations: 8 directions walk + idle + death (11 frames)

─── BRUTUS ───
  Rôle     : ennemi tireur, lent mais dangereux à distance
  Script   : scenes/entities/Brutus.gd
  Hitbox   : CapsuleShape2D radius 24, height 55
  Vitesse  : 55 px/s
  Portée   : 500 px — s'approche puis tire
  Cadence  : 0.6s par balle (BrutusBullet)
  IA       : s'approche du joueur, tire si dans SHOOT_RANGE
             → erre si joueur mort
  HP/dégâts: voir section 2 du doc
  Animations: 8 directions walk + idle + death (spritesheet 3×4 frames)

─── GOLGOTA (boss) ───
  Rôle     : boss final, 3 phases, multiples attaques
  Script   : scenes/entities/Golgota.gd
  HP       : 1000 (à calibrer)
  Vitesse  : 35 / 48 / 60 px/s selon phase
  Phases   : phase 3 (>66% HP), phase 2 (33-66%), phase 1 (<33%)

  ATTAQUES :
  Laser (GolgotaLaser.tscn) :
    - Phase 3 : 1 laser, CD 6s, télégraphe 1.2s, suivi joueur
    - Phase 2 : 3 lasers (central + ±40°), CD 4.5s, télégraphe 0.9s
    - Phase 1 : 5 lasers (central + ±30° + ±65°), CD 3.0s, télégraphe 0.6s
    - Longueur 950px, hitbox height ×2.2 en phase active (0.5s)
    - Dégâts : 25 (laser_damage)

  Orbes (GolgotaOrb.tscn) :
    - Apparaissent en cercle 150-420px autour du boss
    - Attendent 1.8s, puis foncent vers le joueur à 460 px/s
    - Phase 3 : 5 orbes, CD 2.5s | Phase 2 : 8 orbes, CD 2.0s | Phase 1 : 12, CD 1.5s
    - Dégâts : 15

  Contact direct : 30 (damage var, lu par HitArea joueur)

  Spawn d'Aldrich :
    - Phase 2 : 3 Aldrich / 12s autour du joueur
    - Phase 1 : 5 Aldrich / 7s autour du joueur

  Mort : drop BossSoul → boss_souls++, PlayerData.reset_run()
  HUD  : barre de vie violette en bas de l'écran (show_boss_bar / refresh_boss_hp)

9. SHOP
═══════════════════════════════════════════════════════

STRUCTURE : 3 coffres + section upgrades stats + reroll
SCRIPT     : scenes/ui/ShopMenu.gd (extends CanvasLayer)

COFFRES :
  Chaque coffre a une rareté rollée selon le niveau (_roll_rarity())

  Niv.1 : 90%/10%/0%    Niv.5 : 40%/45%/15%  Niv.8 : 10%/35%/55%

  Niv.3 : 70%/25%/5%    Niv.7 : 20%/40%/40%
  Niv.4 : 55%/35%/10%

COÛT EN CLÉS (par rareté Commun/Rare/Épique) :
  Niv.1-3 : 2 / 4 / 6   Niv.4-6 : 3 / 6 / 9   Niv.7-9 : 4 / 8 / 12

REROLL :
  Premier : 30 âmes  Second : 80 âmes  Troisième : 130 âmes  ...

UPGRADES STATS (âmes de run) :
  Coûts : 10 → 25 → 50 → 100 → 200 (5 niveaux max)
  HP +10/niv | Dégâts +1/niv | Vitesse +20/niv | Cadence -0.09s/niv

ÉCONOMIE ÂMES :
  Source : 1 âme/kill + soul_harvester stacks
  Usage  : upgrades shop + reroll
  Reset  : à chaque run (pas sauvegardées)
